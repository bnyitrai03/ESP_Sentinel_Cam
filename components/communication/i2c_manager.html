I2C Manager

The I2C Manager implementation uses the new ESP-IDF I2C master driver, which was introduced in ESP-IDF v5.0.

To the I2C bus the following devices are connected:

  • BQ25622 battery charger

  • OPT3005 ambient light sensor

The I2C pins are connected to the ESP32-S3 as follows:

I2C

ESP32-S3

SDA

GPIO 4

SCL

GPIO 5

Header File

  • components/communication/include/i2c_manager.h

Classes

class I2CManager

I2C Manager class for handling I2C communication. Provides a high-level interface for I2C operations including initialization, read/write operations, and device probing.

Public Functions

I2CManager()

Default constructor. Initializes the I2C manager in a non-initialized state.

~I2CManager()

Destructor. Cleans up I2C driver resources if initialized.

esp_err_t init()

Initialize the I2C driver. Sets up the I2C bus with default configuration (I2C_NUM_0, GPIO 4/5).

Returns:

ESP_OK if successful, otherwise an error code from the ESP-IDF framework.

inline i2c_master_bus_handle_t get_bus_handle()

Get the I2C bus handle.

Returns:

i2c_master_bus_handle_t The handle to the I2C bus.

esp_err_t probe(uint16_t address, int timeout = 5000)

Probe for a device on the I2C bus. Checks if a device with the specified address is connected to the bus.

Parameters:

  • address – The 7-bit I2C address of the device to probe.

  • timeout – Operation timeout in milliseconds (default: 5000ms).

Returns:

ESP_OK if the device is found, otherwise an error code
