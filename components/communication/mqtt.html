MQTT

The camera device communicates with the Sentinel system using this MQTT component.

The communication topics are:

  • imageTopic: MQTT topic for image transmission

  • imageAckTopic: MQTT topic for image header acknowledgment

  • healthReportTopic: MQTT topic for sending the sensor data

  • healthReportRespTopic: MQTT topic for receiving the dynamic config acknowledgment

  • logTopic: MQTT topic for logging

Note

This component requires an active WiFi connection.

esp_err_t publish(const char *topic, const char *data, uint32_t len)

Publishes a message to a specified topic.

Parameters:

  • topic – The topic to publish the message to

  • data – The message data to publish

  • len – The length of the message data

Returns:

  • ESP_OK: the message was published successfully

  • ESP_FAIL: the message failed to publish

bool wait_for_header_ack(const char *expected_timestamp, uint32_t timeout)

Waits for an acknowledgment message with a specific timestamp, which will be sent upon receiving the header message.

Note

This function blocks until the acknowledgment message is received or the timeout is reached

Parameters:

  • expected_timestamp – The expected timestamp for the acknowledgment

  • timeout – The timeout in milliseconds

Returns:

  • true : if the acknowledgment with the expected timestamp is received

  • false : if the acknowledgment is not received within the timeout

bool wait_for_config(uint32_t timeout)

Waits for a new configuration message.

Note

This function blocks until the configuration message is received

Parameters:

timeout – The timeout in milliseconds

Returns:

  • true : if the configuration message is received

  • false : if the configuration message is not received within the timeout
